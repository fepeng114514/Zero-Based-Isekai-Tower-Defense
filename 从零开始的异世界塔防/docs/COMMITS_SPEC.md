# 提交规范
使用约定式提交规范。核心格式如下：

<类型>[可选范围]: <描述>

[可选的正文]

[可选的脚注]

## 常用类型

feat: 新增功能
fix: 修复 Bug
docs: 仅文档变更
style: 代码格式（不影响逻辑，如空格、缩进）
refactor: 代码重构（不修 Bug 也不加功能）
perf: 性能优化
test: 添加或修正测试
chore: 构建流程、工具等变动
ci: CI 配置或脚本修改
build: 影响构建系统或外部依赖

示例 1：
feat(登录): 增加手机号验证码登录方式

- 新增验证码发送接口
- 增加登录校验逻辑

Closes #123

示例 2：
fix: 修复用户未登录时页面白屏的问题
